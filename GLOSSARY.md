# GLOSSARY.md

## Glossary of Terms

This document defines key terms, concepts, and acronyms used throughout the nova-workspace project.

### A

**Agent**  
The AI development assistant that follows AGENT.md rules, maintains logs, and supports self-learning for consistent project progress.

### C

**Context**  
Project-specific information stored in CONTEXT.md that guides the agent's understanding of current state and goals.

### D

**Decision Record**  
A documented architectural or strategic choice recorded in DECISIONS.md with context, rationale, and consequences.

### L

**Log**  
The chronological record of all agent activities, decisions, and outcomes maintained in LOG.md to enable self-improvement and avoid repetition.

### M

**Memory**  
High-level, persistent knowledge base in MEMORY.md that captures long-term insights and strategic information.

### P

**Pattern**  
A proven, reusable solution or convention documented in PATTERNS.md to ensure code and process consistency.

### R

**Requirements**  
Functional and non-functional specifications defined in REQUIREMENTS.md that guide development priorities and validation.

### S

**Self-Learning**  
The agent's ability to analyze LOG.md and MEMORY.md to improve future performance, reduce repetition, and provide better recommendations.

### Key Project Files

- **AGENT.md**: Defines agent behavior and operational rules.
- **ARCHITECTURE.md**: High-level system design and principles.
- **ROADMAP.md**: Strategic development phases and milestones.
- **TODO.md**: Active task tracking and prioritization.

### Acronyms

- **ADR**: Architectural Decision Record
- **CI/CD**: Continuous Integration / Continuous Deployment
- **UI/UX**: User Interface / User Experience

---

This glossary will be expanded as new concepts and terminology are introduced in the project. The agent must reference this file to ensure consistent terminology in all documentation and code comments.
