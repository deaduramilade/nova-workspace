# LOG.md

## Purpose

This log serves as the persistent memory and self-improvement record for the agent. Every significant action, decision, output, and lesson learned must be documented here. The agent **must** read this file at the start of every session (alongside AGENT.md and CONTEXT.md) to avoid repetition, apply learned patterns, and provide better recommendations.

**Logging Rules**:
- Create a new entry for each task or session.
- Use the date and time format: YYYY-MM-DD HH:MM (WAT or UTC).
- Be concise yet comprehensive.
- Highlight reusable patterns, optimizations, and insights for future self-improvement.
- Reference prior entries when relevant to demonstrate continuity.
