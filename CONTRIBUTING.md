# CONTRIBUTING.md

## Welcome

Thank you for considering contributing to nova-workspace. This document outlines the guidelines for contributing to ensure high quality, consistency, and maintainability of the project.

## Development Philosophy

- Prioritize professionalism, reliability, and production readiness.
- Focus on efficiency and minimal resource consumption.
- Maintain clean, well-documented, and secure code.
- Follow established project patterns documented in CONTEXT.md, LOG.md, and MEMORY.md.

## How to Contribute

### 1. Reporting Issues
- Use the GitHub Issues template.
- Provide clear reproduction steps, expected vs actual behavior, and environment details.
- Check existing issues before creating a new one.

### 2. Submitting Pull Requests
- Fork the repository and create a feature branch.
- Ensure changes align with project architecture and conventions.
- Write clear, descriptive commit messages.
- Include tests where applicable.
- Update relevant documentation (README.md, CONTEXT.md, etc.).
- Reference any related issues.

### 3. Code Standards
- Follow the project's coding style and linting rules.
- Maintain professional tone in comments and documentation.
- Ensure all new code is efficient and avoids unnecessary complexity.
- Add or update entries in LOG.md for significant changes when working with the agent.

### 4. Documentation
- Update README.md for new features or setup changes.
- Keep ARCHITECTURE.md and other docs current if they exist.
- All contributions should improve clarity and usability.

## Agent-Assisted Development

When working with the project's AI agent:
- The agent reads AGENT.md, CONTEXT.md, LOG.md, and MEMORY.md on every session.
- Request the agent to log all work in LOG.md.
- Use agent recommendations for optimizations and consistency.

## Pull Request Review Process

- All PRs will be reviewed for code quality, adherence to standards, and alignment with project goals.
- Contributors are expected to address review feedback promptly.
- Maintainers may request changes to ensure enterprise-grade quality.

## Code of Conduct

Please follow standard professional etiquette. We expect respectful and constructive communication from all contributors.

## Questions?

Open a GitHub Discussion or contact the maintainers through appropriate channels.

---

Contributions that follow these guidelines help keep nova-workspace a high-quality, professional open-source project.
